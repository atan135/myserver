# 数据库初始化说明

## 文档定位

本文记录当前仓库 PostgreSQL 初始化与本地重置流程。当前项目还没有线上真实运行库升级需求，数据库结构以 `db/init.sql` 作为唯一初始化来源；历史旧库体系不再作为当前开发目标。

## 当前能力

- `db/init.sql` 是本地开发和空库 bootstrap 的主要初始化脚本，按 `psql` 语义创建各服务数据库与表结构。
- `scripts/reset-dev-data.ps1` 用于清空并重建本地 PostgreSQL 开发库，必须显式传入 `-Confirm`。
- 当前初始化覆盖 `myserver_auth`、`myserver_game`、`myserver_chat`、`myserver_announce` 和 `myserver_mail`。
- 当前不维护迁移状态表、migration runner 或根级数据库迁移脚本。

## 后续范围

以下工作应按 PostgreSQL 继续推进：

- 将 `db/init.sql` 完整拆分为版本化初始 schema migration。
- 为已有开发库、测试库或线上库设计 baseline 标记策略。
- 编写 PostgreSQL 增量升级、回滚和数据修复脚本。
- 将 Node 服务启动期的零散 schema 初始化全部改造成 migration 驱动。

## 当前使用建议

初始化本地空库：

```powershell
psql -U postgres -f db/init.sql
```

清空并重建本地开发库：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\reset-dev-data.ps1 -Confirm
```

`reset-dev-data.ps1` 只处理 PostgreSQL，不清理 Redis、NATS、日志、上传文件、缓存或本地进程。需要完整清理本地开发状态时，应按具体依赖另行处理。

新的业务表开发当前应同步更新 `db/init.sql` 和相关测试。后续出现不能直接重建的持久库后，再恢复 migration 体系并补齐 schema 目录、baseline 规则、升级顺序、回滚/修复策略和 CI 检查命令。
