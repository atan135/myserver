# 资产流水查询与纠正规范

资产流水字段、奖励交付记录、领取恢复和迁移顺序的当前实现边界见[统一资产事务与奖励交付运行边界](../游戏服与接入层/统一资产事务与奖励交付运行边界.md)。

本文约束统一资产事务上线后的流水查询、审计、指标与纠正边界。当前代码优先；本规范不替代资产契约和奖励交付文档。

## 流水边界

`character_asset_ledger` 是 append-only 资产变更证据。每个 grant 行记录请求 ID、角色、物品、绑定摘要、变更前后数量、delta、容器、来源、`origin_type/origin_id`、交付方式与 ID、邮件 ID、fallback 原因、规则版本、快照 revision 和创建时间。紧急纠正额外保存操作者 ID。

流水不保存邮件正文、完整附件、管理 token、请求指纹或完整 JSONB 角色快照。历史 legacy 行以明确的 `legacy` / `legacy-unversioned` 默认值保留，不得伪造为新语义。

奖励因确定满包而转为邮件时，物品尚未进入角色库存，因此不会伪造一条数量变更流水；`reward_mail_outbox` 和 `reward_delivery_records` 保存该邮件交付决定。邮件领取后写入的资产流水会以 `delivery_method=mail`、`mail_id` 和稳定来源关联。

## 查询接口

管理控制面提供只读接口：

`GET /api/v1/assets/ledger`

支持 `character_id`、`request_id`、`origin_type`（兼容 `origin`）、`origin_id`、`delivery_id`、`from`、`to`、`limit` 和 `offset`。请求必须至少带一个定位条件或时间范围；未以角色或请求定位的时间范围最多 31 天，单页最多 100 条。

接口只返回运营排障所需字段，不返回请求指纹、操作者细节、完整绑定对象、原始原因、附件、邮件正文或资产 JSONB。每次成功和失败查询都写入 `admin_audit_logs`，但审计只保存筛选条件和结果数量。

访问权限是独立的 `assets.ledger.read`，当前只授予 `admin` / `super_admin`。管理前端 `/asset-ledger` 仅在拥有该权限时显示，且要求输入至少一个查询条件。

## 紧急纠正

紧急补偿入口为：

`POST /api/v1/gm/emergency-compensate-item`

它要求独立权限 `gm.asset_correction.emergency`，具名 admin actor，以及 game-server 的 `GM_EMERGENCY_ASSET_CORRECTION_ENABLED=true`。请求固定使用 `source=gm-emergency-correction` 和新的 request ID，并通过既有资产事务追加补偿物品；它不提供修改、删除、截断或重写 ledger 的能力。

当前首期纠正方式是 append-only 补偿。若未来需要扣回、冻结或双向资产调整，必须新增带 request ID、操作者、理由和相反 delta 的资产事务，而不能更新历史记录。数据库的 row / truncate trigger 会拒绝直接 `UPDATE` / `DELETE` / `TRUNCATE character_asset_ledger`，这些操作也不属于受支持运维方式。

## 指标与安全审计

game-server 上报以下低基数指标：资产事务平均耗时/次数、revision 冲突、幂等 grant 命中、容量 fallback、奖励邮件 outbox 创建和提交后 push 失败。mail-service 上报奖励邮件创建/幂等重放，以及既有 `blocked_capacity` 速率和 backlog。指标不包含角色、邮件、请求或 token 标签。

admin-api 在单次 GM 发放数量达到 10000，过去 60 秒同管理员同动作达到 10 次，或任何紧急补偿时写入 `security_audit_logs`。安全事件只包含操作者、目标角色、物品 ID、数量、阈值命中情况和时间窗，不包含理由、邮件正文、附件或凭据。

## 部署顺序

先应用 `db/init.sql` 的 additive ledger 列与索引，或先启动包含相同 runtime migration 的 game-server；确认新列存在后再部署 admin-api 和 admin-web。回滚应用时保留 ledger 列和数据，避免让已提交的资产证据失去可读性。后续 schema 清理必须在所有旧应用停止写入后单独评审。
