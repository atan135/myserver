# 角色体系后续功能统一任务清单 Checklist

来源文档：`docs/游戏服与接入层/角色体系与四属性设计.md`

## 目标

统一收拢角色体系 P0-P2 之后仍未完成的后续功能，覆盖角色生命周期接口、背包 / 道具四属性接入、职业 / 流派正式学习、任务 / 成就 / 活动称号联动、场景 / NPC 交互、无等级战斗公式、管理后台 GM 闭环、push 协议、外部客户端同步和最终验收。

本清单不重复拆解已经完成的 P0 身份拆分、P1 四属性基础和 P2 称号与职业阶位基础；这些阶段的完成记录以 `docs/游戏服与接入层/checklists/` 下对应 checklist 为准。服务端角色身份主键迁移已由 `服务端角色身份主键迁移_checklist.md` 单独管理，不纳入本清单。

## 基础原则

- [ ] 本清单新增的角色生命周期、背包、任务、职业、称号、战斗和场景状态默认绑定当前已选角色，不引入新的账号级玩法状态。
- [ ] 所有永久成长变化必须走服务端统一入口，并记录来源、操作者、原因、前后快照和审计日志。
- [ ] 正式玩家接口不得复用 debug token；debug / GM 能力必须有权限、审计和稳定错误码。
- [ ] 阶段内涉及协议、数据库、配置、mock-client、外部客户端和文档时必须同步更新，不保留文档与代码语义不一致的中间状态。
- [ ] 每个阶段完成后运行对应单元测试、协议检查、构建检查和必要的手动验收；真实服务联调或外部客户端联调前先列出依赖并等待用户确认。
- [ ] 不做历史归档设计稿推断；若文档与当前代码冲突，以代码为准，并在阶段内同步修正文档。
