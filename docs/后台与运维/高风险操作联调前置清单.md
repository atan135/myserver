# 高风险操作联调前置清单

## 状态

**尚未执行真实高风险联调，必须在用户明确确认后才可执行。** 本清单只列出受控联调所需依赖、影响和中止条件；不能作为启动服务、连接业务库、发布 NATS 控制事件或执行 GM/rollout 操作的授权。

## 受控环境与身份

- 使用隔离的预发或临时环境，禁止使用开发人员默认端口、共享 token 或生产/业务库。
- 通过 auth schema 中的独立 bootstrap 管理员登录；角色/直接授权、scope、有效期和 Ed25519 私钥必须是该环境专用值。
- 每次写操作必须经 `admin-api` 生成 preflight，再使用同一 `request_id`、nonce、summary hash 和短时 `AdminOperationAssertion` 执行；不得手写 `x-admin-actor`、通用 admin token 或旧内部管理认证。
- 记录操作人、批准人、时间窗、变更单、预期目标、request_id 前缀和回滚负责人。紧急操作另行启用不超过 15 分钟的 break-glass grant。

## 必需依赖

| 依赖 | 联调前必须确认 | 影响与失败处理 |
|------|----------------|----------------|
| Redis | 隔离 namespace 可用；session、限流 Lua、service registry、maintenance key 和 metrics 均指向受控实例 | Redis 不可用时授权/限流 fail-closed；停止联调，不绕过限流或 registry |
| PostgreSQL | 受控 auth/game 库、已应用迁移、备份标识和只追加审计表可写；不使用 `myserver_auth` 等业务库 | 预检、claim 或审计写失败时禁止副作用；结果写失败按 `execution_uncertain` 查询或补偿 |
| Core NATS | 独立 subject/账号、消费者和持久化/监控状态已确认；明确广播、session kick 或 outbox 的订阅范围 | 发布或消费失败按操作结果记录；禁止以重复请求补发造成二次副作用 |
| game-server | 目标实例由 Redis service registry 的 `game-server.admin` endpoint 发现；实例、世界、Room/角色目标和 assertion 公钥已核对 | assertion、expiry、permission、scope、payload hash 或 request_id 校验失败即停止；不可改用固定端口或旧 token |
| game-proxy | 目标实例由 registry 的 `game-proxy.admin` endpoint 发现；rollout/route/maintenance 目标和 assertion 公钥已核对 | 路由或 rollout 未能查询到幂等状态时保留 `execution_uncertain`，不得盲目重试或手工直连 |

## 执行与回滚门槛

1. 先完成只读健康检查、权限 scope 验证和预检，审阅影响摘要、审批状态、备份标识及 recovery plan。
2. 每个高风险写操作单独确认；批量操作必须列出实际目标、最大数量和部分失败处理，禁止通配符扩大范围。
3. 执行后查询操作记录、追加审计、下游 request_id 幂等结果与 NATS 投递结果；发现 `execution_uncertain`、超时、重放或审计失败立即停止后续动作。
4. 可逆操作只使用新的补偿命令，例如解除封禁、关闭维护或恢复已验证版本；不得回写或删除历史审计。不可逆操作须保留备份和人工恢复步骤。
5. 完成后撤销临时 bootstrap grant / break-glass，归档 request_id、审计、监控和验证结果。
